# cooperative_rl

A Q-learning reinforcement learning system for loan repayment optimization in Nigerian cooperative societies. Built as an undergraduate final year project at Achievers University, Owo.

The system learns which intervention to recommend for each borrower — continue their current schedule, restructure the loan, or escalate to an administrator — based on their repayment behavior and financial profile.

## Running the Pipeline

Each module can be run independently. Run them in order the first time.

**1. Generate and save the dataset**

```bash
python src/data_loader.py
```

Creates `data/raw/cooperative_loans.csv` — 5,000 simulated loan records with 13 columns, ~28% default rate.

**2. Verify preprocessing**

```bash
python src/preprocessor.py
```

Encodes categoricals, scales features, applies stratified 70/30 split, balances training set with SMOTE. Should print `X_train shape: (5032, 17)`.

**3. Verify the RL environment**

```bash
python src/rl_environment.py
```

Runs a quick sanity check on state discretization and the reward function.

**4. Train the agent**

```bash
python src/rl_agent.py
```

Trains the Q-learning agent for 1,000 episodes. Prints mean reward for the first and last 100 episodes — you should see improvement.

**5. Run the decision engine**

```bash
python src/decision_engine.py
```

Runs batch recommendations on the test set and prints action distribution. Expected: ~52% Continue, ~32% Restructure, ~16% Flag.

**6. Evaluate**

```bash
python src/evaluator.py
```

Compares Q-learning model against a Logistic Regression baseline. Saves the reward convergence curve to `data/rewards_curve.png`.

**7. Launch the dashboard**

```bash
streamlit run app/dashboard.py
```

Opens the administrator dashboard in your browser.

## Dataset

The dataset is simulated to match Nigerian cooperative lending characteristics, as described in the project report. Key columns:

| Column | Description |
|---|---|
| `age` | Borrower age (25–75), right skewed |
| `monthly_income` | Monthly income in naira (₦50k–₦500k), right skewed |
| `wealth_index` | Ordinal wealth score 1–5, clustered at lower end |
| `loan_amount` | Loan amount in naira (₦50k–₦2M), right skewed |
| `tenure_months` | Loan term in months (6–36), normal around 12 |
| `interest_rate` | Annual interest rate % (5–25), normal around 12% |
| `missed_payments` | Count of missed payments (0–6), zero-inflated |
| `repayment_ratio` | Fraction of loan repaid (0–1), bimodal |
| `outstanding_balance` | Derived: `loan_amount × (1 − repayment_ratio)` |
| `occupation` | civil_servant, trader, teacher, artisan, farmer, other |
| `employment_sector` | public, private, self_employed, informal |
| `default_flag` | Target variable — 1 = default, 0 = repaid (~28% positive) |

## RL Model Design

**State space** — 4 discretized features, each binned into low/medium/high (0/1/2):
- repayment_ratio
- missed_payments
- wealth_index
- outstanding_balance (binned by 33rd/66th percentile of training data)

Total states: 3⁴ = 81. Q-table shape: (81, 3).

**Actions:**
- `0` — Continue current repayment schedule
- `1` — Restructure loan (extend tenure, reduce installment)
- `2` — Flag borrower for administrator follow-up

**Reward:**
- `+1` — repayment_ratio ≥ 0.66 and missed_payments ≤ 1
- `-1` — repayment_ratio < 0.33 or missed_payments ≥ 4
- ` 0` — all other cases

**Hyperparameters:**

| Parameter | Value |
|---|---|
| Learning rate (α) | 0.1 |
| Discount factor (γ) | 0.9 |
| Epsilon start | 1.0 |
| Epsilon end | 0.1 |
| Episodes | 1,000 |
| Steps per episode | 12 (one per repayment period) |

## Evaluation Results

Evaluated on the 1,500-row hold-out test set. The RL model is treated as a binary classifier where FLAG or RESTRUCTURE = predicted default risk.

| Metric | Q-Learning RL | Logistic Regression |
|---|---|---|
| Accuracy | 0.645 | 0.954 |
| Precision | 0.422 | 0.870 |
| Recall | 0.722 | 0.983 |
| F1 Score | 0.533 | 0.923 |
| AUC-ROC | 0.653 | 0.990 |

The Logistic Regression baseline outperforms the RL model on all classification metrics. This is expected — LR was trained directly on the binary default prediction task across all 17 features. The RL model operates on 4 discretized features and wasn't optimized for classification. Its value is in producing actionable interventions, not just a default/no-default label.

The agent's recall of 0.722 means it catches 72% of actual defaulters — useful for a cooperative with no existing early warning system.

## Limitations

- Dataset is simulated, not sourced from a real cooperative. Distributions are calibrated to the literature but not validated against real loan records.
- Q-table covers only 23 of 81 possible states after 1,000 training episodes. Rare borrower profiles map to unvisited states where all Q-values are zero and the agent defaults to ACTION_CONTINUE.
- The evaluation metric gap between the RL model and Logistic Regression reflects a category difference — one optimizes interventions, the other predicts defaults. Direct metric comparison should be interpreted carefully.
- Dashboard runs on localhost only. No authentication, no production deployment.
